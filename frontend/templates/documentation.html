WebAMR Input Settings Documentation

Job Name (Optional)

A user-defined name to help you identify and organize your analysis results. This name will appear on the results page and in the output filenames. This is a WebAMR-specific identifier and does not correspond to an AMRFinderPlus command-line option.

Select an assembled nucleotide file (-n, --nucleotide)

An assembled nucleotide FASTA file. This file must contain the DNA sequence of the organism being analyzed. AMRFinderPlus searches this sequence to identify antimicrobial resistance (AMR) genes using protein-to-nucleotide alignments (blastx). This is highly recommended for accurate detection of AMR genes.

See the official documentation for Input FASTA files.

Select a protein file (-p, --protein)

A protein FASTA file containing amino acid sequences of the genes in the organism. AMRFinderPlus uses this file to identify AMR genes by comparing them to its curated database of known resistance proteins (blastp). Providing a protein file alongside a GFF file allows for more precise coordinate reporting.

See the official documentation for Input FASTA files.

Select a GFF file (-g, --gff)

A GFF3 format file containing gene annotations. This file provides the genomic coordinates and identifiers for the genes in the nucleotide sequence. If you provide a GFF file, you should also provide the corresponding nucleotide and protein files to enable full AMRFinderPlus functionality.

See the official documentation for GFF files.

Annotation Format (--annotation_format)

Specifies the software or pipeline used to generate the gene annotations (GFF and protein files). This is essential for AMRFinderPlus to correctly parse the sequence headers and coordinates. Supported formats include:

  • bakta: From the Bakta annotation pipeline.
  • prokka: From the Prokka annotation pipeline.
  • pgap: From the NCBI Prokaryotic Genome Annotation Pipeline.
  • standard: (default) Use for files downloaded from NCBI resources.
  • ...and others (genbank, microscope, patric, prodigal, pseudomonasdb, rast).

The official documentation for the --annotation_format option contains more information.

Organism group (optional) (-O, --organism)

Specifies the taxonomic group of the organism being analyzed (e.g., Escherichia, Salmonella, Staphylococcus aureus). Providing an organism group allows AMRFinderPlus to use organism-specific curated thresholds and search for known point mutations that confer resistance, significantly improving the accuracy and completeness of the results.

If "None" is selected, AMRFinderPlus will only perform a general search against its core database.

See the official documentation for the --organism option.

File Size and Usage Limits

Currently, WebAMR has a 10 MB file size limit for each upload. This tool is designed to analyze one bacterial genome at a time. Larger submissions or multiple genomes may fail or be rejected.
